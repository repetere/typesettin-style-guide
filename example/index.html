Stylie

Stylie is a barebones css framework generator for professionally designed style guides.

Getting started using stylie

Stylie is designed to generate responsive style guides from professionally designed compositions.

The generated style guides by default are based off of em CSS units, and 12 column, percentage based grids.

Why use Stylie instead of [framework X]

There are a lot of excellent full featured CSS frameworks to pick from; however, if you already posess a fully featured style guide, it quickly becomes extremely tedious to roll back the default styles from larger frameworks.

Many large CSS frameworks also come with JavaScript Dependencies, which adds to the overall weight of your design implementation.

Stylie generated style guides are always namesspaced (by default with [ts]), such that styles may be contained within specific elements and will not conflict with other CSS style definitions

This style guide generating alternative is designed to provide a workstream for generating a comprehensive and bespoke framework suited for your exact specifications.

Methodology

Most classes have six different size class modifiers

  • xs - extra small
  • sm - small
  • md - medium
  • lg - large
  • xl - extra large
  • xx - extra huge

The default size and percent values can be modified by manipulating variables.less

The CSS class names follow the [namespace]-[property]-[value] naming convention. For example:

  • .ts-text-xl - extra large text
  • .ts-text-accent-color - accent colored text
  • .ts-float-right -
    floated right element

The verbose naming convention is intended to make generating composite LESS classes and prototyping HTML element css classes extremely efficient.

By default, the generated style guides are as light weight as possible (no external dependencies, no JavaScript dependencies).

Responsive media queries

The screen[xs:xx] naming convention is used to control the responsive elements by media query.

The default size and percent values can be modified by manipulating variables.less

Visibility

It's possible to hide elements based on the size of the screen

  • .ts-screenxs-hidden - hide element when screen is extra small sized
  • .ts-screensm-hidden - hide element when screen is small sized
  • .ts-screenmd-hidden - hide element when screen is medium sized
  • .ts-screenlg-hidden - hide element when screen is large sized
  • .ts-screenxl-hidden - hide element when screen is extra large sized
  • .ts-screenxx-hidden - hide element when screen is extra huge sized
Grid

You can set the size of a column based on the screen size using the screen[xs:xx] naming convention

  • .ts-screenxs-col-span3 - span 3 columns when screen is extra small sized
  • .ts-screensm-col-span5 - span 5 columns when screen is small sized
  • .ts-screenmd-col-span2 - span 2 columns when screen is medium sized
  • .ts-screenlg-col-span1 - span 1 column when screen is large sized
  • .ts-screenxl-col-span12 - span 12 columns when screen is extra large sized
  • .ts-screenxx-col-span8 - span 8 columns when screen is extra huge sized
Headings

Heading sizes are calculated as ratios based on the size of the screen and the default @heading-size.

  • h1 = 8/8 * h1
  • h2 = 7/8 * h1
  • h3 = 6/8 * h1
  • h4 = 5/8 * h1
  • h5 = 4/8 * h1
  • h6 = 4/8 * h1

Examples:

  • h1 on extra screen - font-size: (@heading-size * @headingxs);
  • h2 on medium screen - font-size: (@heading-size*(7/8) * @headingmd);
  • h3 on extra large screen - font-size: (@heading-size*(6/8) * @headingxl);
  • h4 on small screen - font-size: (@heading-size*(5/8) * @headingsm);
Nav Elements

The nav links will collapse when the screen is smaller than @navcollapse
